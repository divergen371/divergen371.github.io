  • レジスタへの数値の代入・コピーを経て8048478のジャンプ命令で8048491に移動、-0x80(%ebp)ebpと数値0xd(13)を比較する。
  • 8048495のjle命令により-0x80(%ebp)が数値0xdより少ないか、等しい場合に804847aに移動する。
  • 8048481でスタックに格納した数値と-0x7c(%ebp)のxorをとり、-0x40(%ebp)に格納する。

三問目 Remainder theorem ジャンル・PPC

問題

x ≡ 32134 (mod 1584891)
x ≡ 193127 (mod 3438478)

x = ?

フラグはcpaw{xの値}です！

解法

1584891で割ると余り32134、3438478で割ると余り193127、となる整数\(x\)を見つける。

\[a=bq+r (a,bは整数、qは商、rは余り。)\]
から、1つ目の\(x\)は
\[1584891n+32134\tag{1}\label{a}\] 問題の2つの式に共通の\(x\)は \[(1)\div 3438478 = 193127\tag{2}\label{b}\] で求めることができる。
つまり\((1)\)の未知数\(n\)を一つづつ増やして\((2)\)を満たすときの\((1)\)がフラグとなる\(x\)である。
pythonで求める。

未知数\(n\)を一つづつ調べる方法しかないのだろうか？
もっとスマートな解法があったら教えてほしいです。

これは…？

フラグは以下のシンタックスです。
cpaw{復号した値}

※復号した値はそのままで良いですが，実は意味があります，余力がある人は考えてみてください.

解法

これは…?と書かれたリンクに、ヒントとして\(e=13\)を用いた場合の暗号文が提示されているので確認してみる。
ヒントにRSA暗号とあるので問題名と合わせて「Common RSA」でググってみる。
すると、Common Modulus Attackとでてくる。共通法攻撃というそうだ。
いくつかの情報源を読んだ上で以下のように定義されると解釈した。

平文\(m\)を異なるRSA公開鍵\((n,e_1),(n,e_2)\)で暗号化した\(c_1\)と\(c_2\)。
これら共通の\(n\)、それぞれの\(e\)と暗号文があり、且つ\(gcd(e_1,e_2)=1\)のとき
つまり\(e_1\)と\(e_2\)が互いに素となるとき、\(n,e_1,e_2,c_1,c_2\)から平文\(m\)を導くことができる。

これを解くにはツールを自作するか、拾ってくる必要がある。
(ツールの例としてはRsaCtfToolなど。)
ただ今回の問題は\(\sqrt\frac{c_2}{c_1} \)で解けるのでSageMathのように巨大な数を扱える環境さえあればツールを自作する必要はないです。
問題文にある通りフラグはそのまま通るが、とある古代ギリシャ人の歴史家が発明した換字式暗号による暗号文となっている。
